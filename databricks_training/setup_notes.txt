(Side Note: Firewall Setup)
 - In a secure system, connection to ADLS storage account is allowed only from "Selected Virtual Networks and IP addresses"
 - Hence, add both (public and private) Databricks subnets to allowed network
 - ADLS account needs to have a PrivateLink setup in the VNet -> Subnet
 - Firewall rules should allow connection from Databricks subnets or NAT (Network Address Translator) gateway public IP

NAT gateway is the bridge between your private virtual network and Internet
You add a rule in your route table to direct all internet traffic (0.0.0.0/0) through NAT gateway so it take a public IP address

In my setup I added by public IP (https://www.whatismyip.com/) 
192.168.96.73 --> this is a private IP (192.168.x.x, 10.x.x.x, 172.16.x.x - 172.31.x.x)
CIDR : Classless Inter-Domain Routing

(Side Note: Databricks Architecture)
 - Public and Private Subnets
 - Control Plane and Data/Compute plane
The compute plane interacts with the control plane using a public IP address
The control plane lives outside your network so Databricks can manage it efficiently for everyone
(multi-tenant control plane, a shared services layer that Databricks must manage centrally to serve all its customers)
SCIM - System for cross domain identity management (Open Standard)
- For example: Microsoft Entra id users and groups need to be added to Databricks SCIM connector for configuring further access)
SSO - Single Sign On - allows secure access to multiple independent applications using on set of credentials, common protocols - OAuth 2.0, Open ID Connect (OIDC), SAML - Security Assertion Markup Language. It is combined with MFA - MultiFactor Authentication to avoid risk with compromised credentials

SinglePlex Ingestion Model : one-to-one mapping between data sources and bronze tables ) 
- High isolation
- simpler debugging
- easy management of distinct batch datasets

Multiplex Ingestion Model : many-to-one mapping between data sources and bronze table 
- could be limited by the number of parallel processes your system can execute (maximum concurrent job limits)
- suitable for high volume, diverse data
- reduced pipeline complexity
